1. 개인정보의 처리 목적

ST?RO(이하 "회사")는 ST?RO Lite 애플리케이션(이하 "앱")에서 다음 목적을 위해 최소한의 정보를 처리합니다:

  1. 서비스 제공 및 이용자 식별 (익명)
  2. 구독 및 크레딧 잔액 관리
  3. AI 음악 생성 요청 처리
  4. 선물 코드 발급 및 교환
  5. AI 에이전트 환경설정 저장
  6. 음악 생성 진행 상황 알림 (Live Activity / 푸시 알림)
  7. 음악 생성 품질 향상을 위한 익명 통계 분석 (좋아요/싫어요 패턴, 스타일 토큰 분석)
  8. Lab 도구 사용 횟수 관리 및 Lab 크레딧 잔액 관리
  9. Lab AI 도구 기능별 사용 패턴 분석 (익명)
  10. 서비스 개선 및 오류 분석

7. 개인정보의 파기

회사는 개인정보 보유 기간이 경과하거나 처리 목적이 달성된 경우 지체 없이 해당 정보를 파기합니다.

  1. 전자적 파일: 복구 불가능한 방법으로 삭제
  2. Firestore 데이터: 보유 기간 경과 후 일괄 삭제

이용자가 앱을 삭제하면:

  • 기기 내 로컬 데이터 (재생 기록, 설정, 생성된 음악 파일, 채팅 기록)는 즉시 삭제됩니다.
  • 서버 데이터 (Firestore)는 위 보유 기간에 따라 관리됩니다.

8. 이용자의 권리와 행사 방법

이용자는 다음 권리를 행사할 수 있습니다:

  1. 정보 열람 요청: 본인의 수집된 정보 확인
  2. 정보 삭제 요청: 서버에 저장된 데이터 삭제 요청
  3. 구독 취소: iOS 설정 > Apple ID > 구독에서 직접 관리
  4. 구매 복원: 앱 내 구독 페이월에서 "구매 복원" 버튼 사용

권리 행사 요청은 아래 연락처를 통해 접수할 수 있으며, 요청 후 10일 이내에 처리합니다.

9. 개인정보의 안전성 확보 조치

회사는 이용자의 개인정보를 안전하게 보호하기 위해 다음 조치를 취합니다:

  1. 전송 구간 암호화: 모든 통신은 HTTPS(TLS)로 암호화
  2. 서버 보안: Firebase Cloud Run + helmet 보안 헤더 적용
  3. 인증 토큰 검증: 모든 API 요청에 Firebase Auth ID Token 검증
  4. 접근 제어: Firestore 보안 규칙으로 클라이언트 직접 접근 전면 차단 (admin SDK만 허용)
  5. 요청 제한: express-rate-limit으로 비정상 요청 차단 (전역 60/분, 기능별 uid 기반 2~5/분)
  6. 결제 중복 방지: IAP 트랜잭션 ID를 used_transactions 컬렉션에 기록하여 동일 거래 재사용 차단
  7. API 키 관리: 모든 외부 API 키는 Firebase Secrets Manager에 저장 (클라이언트 미노출)
  8. 결제 보안: StoreKit 2 VerificationResult 검증, 미검증 거래 거부
